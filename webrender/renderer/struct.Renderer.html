Used to dispatch functions to the main thread's event loop. Required to allow GLContext sharing in some implementations like WGL.

cache_texture_id_map: Vec<Option<TextureId>>

A vector for fast resolves of texture cache IDs to native texture IDs. This maps to a free-list managed by the backend thread / texture cache. Because of this, items in this array may be None if they have been freed by the backend thread. This saves having to use a hashmap, and allows a flat vector for performance.

external_image_handler: Option<Box<ExternalImageHandler>>

Optional trait object that allows the client application to provide external buffers for image data.

external_images: HashMap<ExternalImageId, TextureId, BuildHasherDefault<FnvHasher>>

Map of external image IDs to native textures.

fn new(options: RendererOptions) -> (Renderer, RenderApiSender)

Initializes webrender and creates a Renderer and RenderApiSender.

Examples

Initializes a Renderer with some reasonable values. For more information see RendererOptions.

let opts = webrender::RendererOptions {
   device_pixel_ratio: 1.0,
   resource_override_path: None,
   enable_aa: false,
   enable_msaa: false,
   enable_profiler: false,
};
let (renderer, sender) = Renderer::new(opts);

fn set_render_notifier(&self, notifier: Box<RenderNotifier>)

Sets the new RenderNotifier.

The RenderNotifier will be called when processing e.g. of a (scrolling) frame is done, and therefore the screen should be updated.

fn set_flush_notifier(&self, flush_notifier: Box<FlushNotifier>)

Sets the new FlushNotifier

The FlushNotifier is called after all messages to the backend thread have been processed. This should be used when we need to sync wait for rendering to happen, such as with reftests.
