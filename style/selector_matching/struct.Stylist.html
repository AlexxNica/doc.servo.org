This structure holds all the selectors and device characteristics for a given document. The selectors are converted into Rules (defined in rust-selectors), and introduced in a SelectorMap depending on the pseudo-element (see PerPseudoElementSelectorMap), and stylesheet origin (see the fields of PerPseudoElementSelectorMap).

This structure is effectively created once per pipeline, in the LayoutThread corresponding to that pipeline.

Fields

device: Device

Device that the stylist is currently evaluating against.

viewport_constraints: Option<ViewportConstraints>

Viewport constraints based on the current device.

quirks_mode: bool

If true, the quirks-mode stylesheet is applied.

is_device_dirty: bool

If true, the device has changed, and the stylist needs to be updated.

element_map: PerPseudoElementSelectorMap

The current selector maps, after evaluating media rules against the current device.

pseudos_map: FnvHashMap<PseudoElement, PerPseudoElementSelectorMap>

The selector maps corresponding to a given pseudo-element (depending on the implementation)

animations: FnvHashMap<Atom, KeyframesAnimation>

A map with all the animations indexed by name.

precomputed_pseudo_element_decls: FnvHashMap<PseudoElement, Vec<ApplicableDeclarationBlock>>

Applicable declarations for a given non-eagerly cascaded pseudo-element. These are eagerly computed once, and then used to resolve the new computed values on the fly on layout.

rules_source_order: usize state_deps: DependencySet

Selector dependencies used to compute restyle hints.

sibling_affecting_selectors: Vec<Selector<TheSelectorImpl>>

Selectors in the page affecting siblings

non_common_style_affecting_attributes_selectors: Vec<Selector<TheSelectorImpl>>

Selectors in the page matching elements with non-common style-affecting attributes.
