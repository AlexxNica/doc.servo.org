A Pipeline is the constellation's view of a Document. Each pipeline has an event loop (executed by a script thread) and a layout thread. A script thread may be responsible for many pipelines, but a layout thread is only responsible for one.

Fields

id: PipelineId

The ID of the pipeline.

frame_id: FrameId

The ID of the frame that contains this Pipeline.

parent_info: Option<(PipelineId, FrameType)>

The parent pipeline of this one. None if this is a root pipeline. Note that because of mozbrowser iframes, even top-level pipelines may have a parent (in which case the frame type will be MozbrowserIFrame). TODO: move this field to Frame.

event_loop: Rc<EventLoop>

The event loop handling this pipeline.

layout_chan: IpcSender<LayoutControlMsg>

A channel to layout, for performing reflows and shutdown.

compositor_proxy: Box<CompositorProxy + Send + 'static>

A channel to the compositor.

url: ServoUrl

The most recently loaded URL in this pipeline. Note that this URL can change, for example if the page navigates to a hash URL.

title: Option<String>

The title of the most recently-loaded page.

size: Option<TypedSize2D<f32, PagePx>>

The size of the frame. TODO: move this field to Frame.

running_animations: bool

Whether this pipeline is currently running animations. Pipelines that are running animations cause composites to be continually scheduled.

children: Vec<FrameId>

The child frames of this pipeline (these are iframes in the document).

is_private: bool

Whether this pipeline is in private browsing mode. TODO: move this field to Frame.

visible: bool

Whether this pipeline should be treated as visible for the purposes of scheduling and resource management.
